2.配置，编译
    ①kernel等复杂项目，都不能直接编译，都要先配置才能编译
	  uboot也要先配置，配置方法是:cd进入uboot根目录，然后在根目录下执行
	  make x210_sd_config
	  出现Configuring for x210_sd board...则表示配置成功
	②编译
	  注意安装了arm-linux-gcc工具链
	  检查当前目录的Makefile中编译器的设置是否正确。在工程的总Makefile中设置的交叉
	  编译工具链的路径和名字必须和我们的路径和名字一致。
	  /usr/local/arm/arm-2009q3/bin/arm-none-linux-gnueabi-
	  确保以上可以编译，make即可make -j4(多线程编译)make -j2(多线程编译)电脑多核
3.uboot的源码目录分析
     以九鼎官方的uboot为基本学习，三星官方对照
	 ①文件介绍
	   .gitignore   git工具的文件，git是一个版本管理工具，和uboot本身无关
	   arm_config.mk  是一个Makefile文件,在别的Makefile会调用
	   三个Changelog文件，修改记录文件
	   config.mk 
	   COPYING    版权声明
	   CREDITS(credits)鸣谢，对uboot有贡献的人
	   image_split 一个脚本
	   MAINTAINERS（maintainers）维护者
       MAKEALL  一个脚本,应该是帮助编译uboot	
       Makefile  uboot源代码的主Makefile
       mk 一个脚本，里面是快速编译(清理，配置，编译)
       mkconfig这个很重要，是uboot配置阶段的主要配置脚本
       mkmovi
	   README
	   rule.mk  uboot的Makefile使用规则
	   以上文件重要的是mkconfig  Makefile
	  ②文件夹
	    api文件夹  硬件无关功能函数的API
		api_examples API的测试示例代码
		board 文件夹下每一个文件代表一个开发板，内容描述开发板的信息
		common 与具体硬件无关的，普遍使用的代码，比如控制台的实现，crc校验，更多的
		主要是两类，cmd开头用来实现uboot的命令系统。env开头，用来实现环境变量
	    cpu  Soc相关的初始化和控制代码(譬如cpu的中断,串口也包括起始代码start.S)
		disk  和磁盘有关
		doc   文档目录
		drivers  驱动，linux的设备驱动从liunx源代码移过来的，如网卡，flash，
		inand，nandflash
		examples  示例代码
		fs  (filesyatem)文件系统，从linux移植过来的，管理flash
		include 头文件目录
		lib_开头的(典型的lib_arm，lib_generic)架构相关的库文件。lib_arm是arm架构的
		使用的一些库文件，lib_generic是所有架构通用的库文件
		libfdt  设备树有关的
		nand_spl nand相关的
		net 网络相关的，譬如uboot的tftp，net，ping
		onenand
		sd_fusing 刷写SD卡，需要仔细研究
		tools  工具类的代码
		board，common，cpu，drivers，include，sd_fusing比较重要
